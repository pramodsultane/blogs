Modular AWS VPC Architecture with Terraform

Build reusable, production-ready VPC modules with multi-tier subnets, NAT gateways, flow logs, and consistent tagging — deployable across any region in minutes.

PK
Pramod
Cloud & DevOps Engineer
| 📅 1 March 2026 | 📖 9 min read

A well-designed VPC is the foundation of any secure, scalable AWS workload. Yet most teams copy-paste one-off VPC configs that become impossible to maintain at scale. In this post I'll walk through a reusable Terraform module that provisions a production-grade VPC with multi-tier subnets, NAT gateways, VPC flow logs, and sensible defaults — ready to drop into any environment with a single module call.

Architecture Overview #

The module creates three subnet tiers across configurable Availability Zones:

  • Public subnets — Load balancers, NAT gateways, bastion hosts
  • Private subnets — Application servers, EKS nodes, Lambda functions
  • Isolated subnets — RDS, ElastiCache, MSK — no internet access whatsoever
💡

The module assumes a /16 CIDR block and auto-calculates all subnet CIDRs using cidrsubnet(), so you never have to do the math manually.

TierInternet AccessRoute TableTypical Resources
PublicDirect (IGW)→ Internet GatewayALB, NAT GW, Bastion
PrivateOutbound via NAT→ NAT GatewayEKS, EC2, Lambda
IsolatedNoneLocal onlyRDS, ElastiCache, MSK

Module File Structure #

directory layout

    ├── main.tf          # Resources: VPC, subnets, gateways, routes

variables.tf #

modules/vpc/variables.tf 
variable "name" {
  description = "Name prefix applied to all resources"
  type        = string
}

variable "cidr_block" {
  description = "CIDR block for the VPC (e.g. 10.0.0.0/16)"
  type        = string
  default     = "10.0.0.0/16"
}

variable "azs" {
  description = "List of Availability Zones to deploy subnets into"
  type        = list(string)
  # Example: ["us-east-1a", "us-east-1b", "us-east-1c"]
}

variable "enable_nat_gateway" {
  description = "Provision a NAT Gateway for private subnet egress"
  type        = bool
  default     = true
}

variable "single_nat_gateway" {
  description = "Use one NAT GW for all AZs (saves cost in non-prod)"
  type        = bool
  default     = false
}

variable "enable_flow_logs" {
  description = "Enable VPC Flow Logs to CloudWatch"
  type        = bool
  default     = true
}

variable "tags" {
  description = "Additional tags merged into all resources"
  type        = map(string)
  default     = {}
}

main.tf #

VPC, Subnets & Gateways

modules/vpc/main.tf 
locals {
  az_count      = length(var.azs)
  common_tags   = merge({ Module = "vpc", Name = var.name }, var.tags)
}

# ── VPC ──────────────────────────────────────────────────────────
resource "aws_vpc" "this" {
  cidr_block           = var.cidr_block
  enable_dns_support   = true
  enable_dns_hostnames = true
  tags                 = merge(local.common_tags, { Name = "${var.name}-vpc" })
}

# ── Internet Gateway ─────────────────────────────────────────────
resource "aws_internet_gateway" "this" {
  vpc_id = aws_vpc.this.id
  tags   = merge(local.common_tags, { Name = "${var.name}-igw" })
}

# ── Public Subnets ───────────────────────────────────────────────
resource "aws_subnet" "public" {
  count                   = local.az_count
  vpc_id                  = aws_vpc.this.id
  cidr_block              = cidrsubnet(var.cidr_block, 4, count.index)
  availability_zone       = var.azs[count.index]
  map_public_ip_on_launch = true
  tags = merge(local.common_tags, {
    Name = "${var.name}-public-${var.azs[count.index]}"
    Tier = "public"
  })
}

# ── Private Subnets ──────────────────────────────────────────────
resource "aws_subnet" "private" {
  count             = local.az_count
  vpc_id            = aws_vpc.this.id
  cidr_block        = cidrsubnet(var.cidr_block, 4, count.index + local.az_count)
  availability_zone = var.azs[count.index]
  tags = merge(local.common_tags, {
    Name = "${var.name}-private-${var.azs[count.index]}"
    Tier = "private"
  })
}

# ── Isolated Subnets ─────────────────────────────────────────────
resource "aws_subnet" "isolated" {
  count             = local.az_count
  vpc_id            = aws_vpc.this.id
  cidr_block        = cidrsubnet(var.cidr_block, 4, count.index + local.az_count * 2)
  availability_zone = var.azs[count.index]
  tags = merge(local.common_tags, {
    Name = "${var.name}-isolated-${var.azs[count.index]}"
    Tier = "isolated"
  })
}

# ── NAT Gateways ─────────────────────────────────────────────────
resource "aws_eip" "nat" {
  count  = var.enable_nat_gateway ? (var.single_nat_gateway ? 1 : local.az_count) : 0
  domain = "vpc"
  tags   = merge(local.common_tags, { Name = "${var.name}-nat-eip-${count.index}" })
}

resource "aws_nat_gateway" "this" {
  count         = var.enable_nat_gateway ? (var.single_nat_gateway ? 1 : local.az_count) : 0
  allocation_id = aws_eip.nat[count.index].id
  subnet_id     = aws_subnet.public[count.index].id
  tags          = merge(local.common_tags, { Name = "${var.name}-nat-${count.index}" })
  depends_on    = [aws_internet_gateway.this]
}

# ── Route Tables ─────────────────────────────────────────────────
resource "aws_route_table" "public" {
  vpc_id = aws_vpc.this.id
  route {
    cidr_block = "0.0.0.0/0"
    gateway_id = aws_internet_gateway.this.id
  }
  tags = merge(local.common_tags, { Name = "${var.name}-rt-public" })
}

resource "aws_route_table" "private" {
  count  = var.enable_nat_gateway ? (var.single_nat_gateway ? 1 : local.az_count) : local.az_count
  vpc_id = aws_vpc.this.id
  dynamic "route" {
    for_each = var.enable_nat_gateway ? [1] : []
    content {
      cidr_block     = "0.0.0.0/0"
      nat_gateway_id = aws_nat_gateway.this[var.single_nat_gateway ? 0 : count.index].id
    }
  }
  tags = merge(local.common_tags, { Name = "${var.name}-rt-private-${count.index}" })
}

# ── Route Table Associations ─────────────────────────────────────
resource "aws_route_table_association" "public" {
  count          = local.az_count
  subnet_id      = aws_subnet.public[count.index].id
  route_table_id = aws_route_table.public.id
}

resource "aws_route_table_association" "private" {
  count          = local.az_count
  subnet_id      = aws_subnet.private[count.index].id
  route_table_id = aws_route_table.private[var.single_nat_gateway ? 0 : count.index].id
}

resource "aws_route_table_association" "isolated" {
  count          = local.az_count
  subnet_id      = aws_subnet.isolated[count.index].id
  route_table_id = aws_vpc_main_route_table_id.this  # implicit local-only route
}

VPC Flow Logs

modules/vpc/main.tf (flow logs section) 
resource "aws_cloudwatch_log_group" "flow_logs" {
  count             = var.enable_flow_logs ? 1 : 0
  name              = "/aws/vpc/flow-logs/${var.name}"
  retention_in_days = 30
  tags              = local.common_tags
}

resource "aws_iam_role" "flow_logs" {
  count = var.enable_flow_logs ? 1 : 0
  name  = "${var.name}-vpc-flow-logs-role"
  assume_role_policy = jsonencode({
    Version = "2012-10-17"
    Statement = [{
      Effect    = "Allow"
      Principal = { Service = "vpc-flow-logs.amazonaws.com" }
      Action    = "sts:AssumeRole"
    }]
  })
}

resource "aws_iam_role_policy_attachment" "flow_logs" {
  count      = var.enable_flow_logs ? 1 : 0
  role       = aws_iam_role.flow_logs[0].name
  policy_arn = "arn:aws:iam::aws:policy/CloudWatchLogsFullAccess"
}

resource "aws_flow_log" "this" {
  count           = var.enable_flow_logs ? 1 : 0
  vpc_id          = aws_vpc.this.id
  traffic_type    = "ALL"
  iam_role_arn    = aws_iam_role.flow_logs[0].arn
  log_destination = aws_cloudwatch_log_group.flow_logs[0].arn
}
⚠️

Cost note: NAT Gateways are charged per hour (~$0.045/hr) plus per-GB data processing. Use single_nat_gateway = true in dev/staging to reduce costs. Never use a single NAT GW in production — it creates an AZ-level single point of failure.

outputs.tf #

modules/vpc/outputs.tf 
output "vpc_id" {
  description = "ID of the created VPC"
  value       = aws_vpc.this.id
}

output "vpc_cidr_block" {
  value = aws_vpc.this.cidr_block
}

output "public_subnet_ids" {
  description = "IDs of the public subnets"
  value       = aws_subnet.public[*].id
}

output "private_subnet_ids" {
  description = "IDs of the private subnets"
  value       = aws_subnet.private[*].id
}

output "isolated_subnet_ids" {
  description = "IDs of the isolated (DB) subnets"
  value       = aws_subnet.isolated[*].id
}

output "nat_gateway_ids" {
  value = aws_nat_gateway.this[*].id
}

Using the Module #

environments/prod/main.tf 
module "vpc" {
  source = "../../modules/vpc"

  name       = "myapp-prod"
  cidr_block = "10.10.0.0/16"
  azs        = ["us-east-1a", "us-east-1b", "us-east-1c"]

  enable_nat_gateway = true
  single_nat_gateway = false   # one NAT GW per AZ for HA
  enable_flow_logs   = true

  tags = {
    Environment = "production"
    Owner       = "platform-team"
    CostCenter  = "engineering"
  }
}

# Use outputs in other resources
resource "aws_eks_cluster" "this" {
  name     = "myapp-prod"
  role_arn = aws_iam_role.eks.arn

  vpc_config {
    subnet_ids = module.vpc.private_subnet_ids
  }
}

Best Practices & Tips #

Tagging Strategy

Consistent tagging is non-negotiable for cost allocation and automation. Always include at minimum: Environment, Owner, CostCenter, and ManagedBy = "terraform". The module enforces ManagedBy automatically via local.common_tags.

CIDR Planning

A /16 gives you 65,536 IPs split into subnets with a /4 offset via cidrsubnet(). For 3 AZs this produces /20 subnets (~4,000 IPs each) — ample for most workloads. Adjust the newbits argument if you need smaller or larger blocks.

State Backend

versions.tf 
terraform {
  required_version = ">= 1.6"

  required_providers {
    aws = {
      source  = "hashicorp/aws"
      version = "~> 5.0"
    }
  }

  backend "s3" {
    bucket         = "myapp-terraform-state"
    key            = "vpc/terraform.tfstate"
    region         = "us-east-1"
    encrypt        = true
    dynamodb_table = "terraform-state-lock"
  }
}
✅

Tip: Use terraform plan -out=tfplan and always review the plan before applying network changes. A destroy on an active VPC with running workloads will fail gracefully but can leave orphaned ENIs — always deprovision workloads first.

Conclusion #

With this module pattern you define your VPC once and deploy it consistently across every environment. The key wins are: no hardcoded CIDRs, HA NAT gateways with a cost-saving override for non-prod, automatic flow logs for security auditing, and clean outputs that plug straight into EKS, RDS, and ALB module calls.

Next steps — wire the private_subnet_ids output into an EKS node group or an RDS subnet group, and add a vpc_endpoints block to route S3 and DynamoDB traffic over the AWS backbone instead of the NAT gateway, which will cut data-transfer costs significantly.
